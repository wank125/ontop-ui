# 天织 Ontop UI 使用说明书

本文档基于仓库当前代码整理，覆盖 `ontop-ui` 的真实运行方式、功能模块、操作流程、接口能力和已知限制。

## 1. 系统概述

`ontop-ui` 是一个基于 Ontop 的虚拟知识图谱工作台，目标是把关系数据库通过 Ontop 映射为可查询的语义层，并提供以下能力：

- 数据源管理
- 数据库结构探查
- Ontop Bootstrap 生成本体与映射
- OBDA 映射文件编辑
- SPARQL 查询与 SQL 重写查看
- AI 自然语言问答
- 本体可视化与 TTL 解析
- 系统与模型配置查看

系统由三部分组成：

- 前端：Next.js 16 + React 19 + TypeScript
- 后端：FastAPI + SQLite + OpenAI SDK
- 语义引擎：Ontop CLI 5.5.0 + Ontop Endpoint

## 2. 目录说明

核心代码集中在以下目录：

- `ontop-ui/backend`：FastAPI 后端、SQLite、Ontop 调用、AI 接口
- `ontop-ui/frontend`：前端页面、API 代理、可视化界面
- `ontop-ui/docker`：前后端与 PostgreSQL 的 Docker 构建资源
- `ontop-ui/docs`：说明文档、截图、测试计划

关键入口文件：

- `backend/main.py`：后端入口
- `backend/config.py`：运行配置
- `frontend/src/server.ts`：前端 Node 服务入口，同时代理 `/api/*`
- `docker-compose.yml`：默认 retail 演示环境
- `docker-compose.lvfa.yml`：LVFA 演示环境

## 3. 运行前准备

### 3.1 依赖要求

- Python 3.10+
- Node.js 20+
- `pnpm >= 9`
- Java 11+（Ontop 运行需要）
- Docker / Docker Compose（推荐）

### 3.2 外部依赖

系统运行依赖以下外部资源：

- PostgreSQL 数据库
- Ontop CLI 可执行文件
- Ontop 输出目录（存放 `.ttl`、`.obda`、`.properties`）
- 可选 LLM 服务，要求兼容 OpenAI Chat Completions

支持的 LLM Provider：

- OpenAI
- LM Studio
- Ollama
- DeepSeek
- 智谱
- Azure OpenAI
- Anthropic
- 自定义 OpenAI 兼容服务

## 4. 启动方式

### 4.1 Docker 启动

#### 默认 retail 环境

在 `ontop-ui` 目录执行：

```bash
docker compose up -d --build
```

访问地址：

- 前端：`http://localhost:3000`
- 后端 OpenAPI：`http://localhost:8000/docs`
- PostgreSQL：`localhost:5435`

#### LVFA 演示环境

```bash
docker compose -f docker-compose.lvfa.yml up -d --build
```

访问地址：

- 前端：`http://localhost:3001`
- 后端 OpenAPI：`http://localhost:8001/docs`
- PostgreSQL：`localhost:5436`

说明：

- Docker 中前端容器内部监听 `5000`
- Docker 中后端容器内部监听 `8000`
- 前端通过 `BACKEND_URL` 访问后端

## 5. 环境变量

后端主要环境变量定义于 `backend/config.py`。

### 5.1 Ontop 相关

- `ONTOP_BASE`：Ontop 测试目录根路径
- `ONTOP_CLI`：Ontop CLI 可执行文件路径
- `ONTOP_OUTPUT`：输出目录
- `ONTOLOGY_FILE`：默认本体文件路径
- `MAPPING_FILE`：默认映射文件路径
- `PROPERTIES_FILE`：默认 JDBC 属性文件路径

默认本地值会指向仓库中的：

- `../ontop-test/ontop-cli/ontop`
- `../ontop-test/output`

### 5.2 服务相关

- `ONTOP_ENDPOINT_URL`：默认 `http://localhost:8080`
- `ONTOP_ENDPOINT_PORT`：默认 `8080`
- `FASTAPI_PORT`：默认 `8000`

### 5.3 LLM 相关

- `LLM_BASE_URL`：默认 `http://localhost:1234/v1`
- `LLM_API_KEY`：默认 `lm-studio`
- `LLM_MODEL`：默认 `zai-org/glm-4.7-flash`

### 5.4 安全相关

- `ENCRYPTION_KEY`：可选，用于覆盖 SQLite 中敏感字段加密密钥

如果不提供，系统会自动在 `backend/data/.encryption_key` 生成 Fernet 密钥。

## 6. 系统数据存储

### 6.1 SQLite

后端启动时会自动初始化 SQLite：

- 数据库文件：`backend/data/ontop_ui.db`

表结构包括：

- `datasources`：数据源配置
- `ai_config`：AI 模型配置
- `query_history`：SPARQL 查询历史

其中以下信息会加密存储：

- 数据源密码
- `llm_api_key`

### 6.2 文件系统输出

Bootstrap 和 Ontop 相关文件保存在输出目录中：

- `.ttl`：本体文件
- `.obda`：映射文件
- `.properties`：JDBC 属性文件
- `manifest.json`：Bootstrap 输出元数据
- `selected_tables.json`：局部 Bootstrap 所选表信息

如果从页面触发局部或全量 Bootstrap，后端会生成按时间戳命名的版本目录。

## 7. 功能模块说明

### 7.1 首页 `/`

首页是工作台导航页，不负责实际数据处理。

主要功能：

- 展示数据源数量、映射数量、本体类数量、查询历史数量
- 展示 Ontop 端点运行状态
- 展示当前 AI Provider、模型、快捷问题数量
- 提供到各模块的快捷入口

### 7.2 登录页 `/login`

当前登录页为前端演示页，不接入真实认证系统。

实际行为：

- 账号默认预填
- 只要密码非空即可进入首页
- 用户信息为前端 mock 数据

因此当前版本更适合本地演示和功能联调，不适合作为正式认证入口。

### 7.3 数据源管理 `/datasource`

功能：

- 创建数据源
- 删除数据源
- 测试连接
- 读取数据库结构
- 执行全量 Bootstrap

支持数据库驱动：

- PostgreSQL
- MySQL
- Oracle
- SQL Server

创建数据源时需要填写：

- 名称
- JDBC URL
- 用户名
- 密码
- 驱动类型

常用操作流程：

1. 创建数据源
2. 点击“测试连接”
3. 点击“探测结构”
4. 点击“Bootstrap”

页面会显示 Bootstrap 产物路径：

- `ontologyPath`
- `mappingPath`
- `propertiesPath`

说明：

- 本页触发的 Bootstrap 为全量模式
- `base_iri` 固定使用 `http://example.com/ontop/`

### 7.4 数据库概览 `/db-schema`

功能：

- 从已配置数据源读取 schema
- 浏览表、列、主键、唯一约束、外键
- 搜索表名
- 选择部分表执行局部 Bootstrap
- 预览 Bootstrap 影响面

局部 Bootstrap 支持：

- 选择一个或多个表
- 自动补齐依赖表
- 查看预估类与对象属性
- 生成版本化输出目录

后端逻辑说明：

- 如果 `mode=partial` 或传入 `tables`，则执行局部 Bootstrap
- 局部 Bootstrap 会先跑一次全量 Ontop Bootstrap
- 再根据选定表过滤映射，并重新生成裁剪后的本体

适用场景：

- 只想针对局部业务域生成本体
- 先做小范围建模验证

### 7.5 映射编辑 `/mapping`

功能：

- 列出输出目录中的 `.obda` 文件
- 解析前缀声明与映射规则
- 新增映射
- 编辑映射
- 删除映射
- 保存映射文件
- 验证映射
- 重启 Ontop Endpoint

页面视图包括：

- 文件选择
- Prefix 区域
- Mapping 列表
- 选中 Mapping 的目标类、来源表和属性数量摘要

验证逻辑：

- 通过 `ontop validate` 调用 Ontop CLI

端点重启逻辑：

- 调用后端 `start_endpoint`
- 使用默认或传入的 `ontology_path / mapping_path / properties_path`

说明：

- 页面支持结构化编辑，不提供原始文本编辑器
- 文件必须已存在于输出目录中

### 7.6 SPARQL 查询 `/sparql`

功能：

- 编辑并执行 SPARQL
- 表格方式显示结果
- 查看 Ontop SQL 重写结果
- 查看历史查询

执行流程：

1. 前端调用 `/api/v1/sparql/query`
2. 后端将查询转发到 Ontop Endpoint `/sparql`
3. 成功后写入查询历史
4. 前端再调用 `/api/v1/sparql/reformulate` 获取 SQL

查询历史特点：

- 保存在 SQLite
- 最多保留 100 条
- 页面右侧可点击历史查询回填到编辑器

### 7.7 AI 助手 `/ai-assistant`

功能：

- 输入自然语言问题
- SSE 流式展示执行步骤
- 查看生成出的 SPARQL、SQL 和结果文本
- 最终生成中文答案

后端执行链路：

1. 读取当前映射文件，抽取类、属性、关系与前缀
2. 根据系统提示词构造 SPARQL 生成提示
3. 调用 LLM 生成 SPARQL
4. 自动补全缺失的 `PREFIX`
5. 调用 Ontop Endpoint 获取 SQL 改写与查询结果
6. 再调用 LLM 生成最终自然语言回答

流式事件类型包括：

- `analyzing`
- `sparql_generated`
- `executing`
- `executed`
- `answer`

注意：

- 当前 AI 页面里的快捷问题是页面内置常量
- 设置页保存的“快捷问题”当前没有接入此页面读取逻辑

### 7.8 本体可视化 `/ontology`

功能：

- 列出输出目录中的 `.obda` 与 `.ttl` 文件
- 图谱模式下根据 `.obda` 提取类和关系
- 定义模式下解析 `.ttl`，展示类、对象属性、数据属性和 SHACL 约束

图谱能力：

- 缩放
- 拖拽
- 自适应视图
- 基于 Vis.js 的关系网络展示

TTL 解析能力：

- 解析中英文标签
- 解析类注释
- 解析对象属性、数据属性
- 解析 SHACL property 与 SPARQL 约束

### 7.9 AI 设置 `/settings`

功能：

- 选择 Provider
- 填写 Base URL、API Key、Model、Temperature、Max Tokens
- 自动发现模型列表
- 编辑系统提示词
- 维护快捷问题列表

模型发现逻辑：

- OpenAI 兼容服务：调用 `/models`
- Ollama：调用 `/api/tags`
- 拉取失败时回退到预设模型列表或手工输入

保存后端行为：

- 配置写入 SQLite
- 自动重新加载 LLM 客户端

### 7.10 系统设置 `/system`

功能：

- 显示 mock 用户信息
- 显示后端健康状态
- 显示 Ontop 端点状态
- 显示只读系统配置

显示内容包括：

- Ontop CLI 路径
- Ontop 端点地址
- LLM API 地址
- LLM 模型

## 8. 推荐使用流程

建议按以下顺序使用：

1. 进入 `/datasource` 创建并测试数据源
2. 进入 `/db-schema` 查看结构并决定是否做局部 Bootstrap
3. 在 `/mapping` 检查和修正自动生成的 OBDA 规则
4. 在 `/mapping` 中验证映射并重启端点
5. 在 `/sparql` 手工验证查询
6. 在 `/settings` 配置好 LLM
7. 在 `/ai-assistant` 做自然语言问答
8. 在 `/ontology` 查看图谱和本体定义

## 10. 日志与排障

日志目录：

- `ontop-ui/logs/backend.log`
- `ontop-ui/logs/frontend.log`

常见问题排查建议：

- 前端打不开：确认前端实际端口是否为 `5000`、`3000` 或 `3001`
- 后端接口失败：查看 `backend.log`
- SPARQL 无法执行：检查 Ontop Endpoint 是否已启动
- Bootstrap 失败：检查 JDBC 信息、数据库权限和 Ontop CLI 路径
- AI 查询失败：检查 `LLM_BASE_URL`、`LLM_API_KEY`、模型名和服务连通性

## 11. 已知限制

以下内容来自当前代码实际状态：

- 登录页不是正式认证，仅为演示入口
- 系统页展示的用户信息是 mock 数据
- AI 助手页的快捷问题未接入设置页保存结果
- 数据源页只支持全量 Bootstrap；局部 Bootstrap 在数据库概览页执行
- Ontop Endpoint 需要显式启动或重启后，SPARQL/AI 查询才可正常工作
- 前端 README 与实际启动脚本端口不完全一致，应以脚本和 compose 文件为准
