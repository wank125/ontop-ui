# 天织 Ontop UI 功能设计文档

## 1. 文档目的

本文档用于说明 `ontop-ui` 当前版本的功能范围、目标用户、页面设计、交互流程、接口依赖和功能约束，作为产品设计、研发实现和测试验收的共同依据。

文档基于当前代码和页面实现整理，不包含尚未落地的规划功能。

## 2. 产品定位

天织 Ontop UI 是一个面向语义建模和虚拟知识图谱管理的工作台，目标是围绕 Ontop 提供一条完整操作链路：

- 接入数据库
- 理解数据库结构
- 自动生成本体与映射
- 调整映射规则
- 执行 SPARQL
- 通过 AI 做自然语言查询
- 以图谱和定义视角查看本体

## 3. 目标用户

当前版本主要面向以下用户：

- 数据建模人员
- 语义层开发人员
- 数据平台工程师
- 本体原型验证人员
- 演示与售前方案人员

## 4. 功能范围

当前版本已实现模块：

- 首页
- 登录页
- 数据源管理
- 数据库概览
- 映射编辑
- SPARQL 查询
- AI 助手
- 本体可视化
- AI 设置
- 系统设置

不在当前版本范围内：

- 多用户权限控制
- 正式账号认证
- 工作流审批
- 文件版本回滚
- 完整审计日志页面
- 多租户隔离

## 5. 用户角色与权限假设

当前版本默认单一角色视角：

- 管理员 / 建模者

行为特征：

- 可以访问所有页面
- 可以创建删除数据源
- 可以修改映射和 AI 配置
- 可以重启 Ontop Endpoint

说明：

- 当前代码中没有真实权限控制
- 登录页和系统页展示的用户信息是 mock 数据

## 6. 用户任务主线

核心任务主线如下：

1. 配置数据源
2. 测试连接并读取 schema
3. 执行全量或局部 Bootstrap
4. 查看和修改映射规则
5. 验证映射并启动端点
6. 执行 SPARQL 查询
7. 配置 AI 模型
8. 用自然语言提问
9. 查看本体图谱和定义

## 7. 页面功能设计

### 7.1 首页 `/`

#### 目标

向用户说明平台能力，并快速引导到正确功能页。

#### 主要内容

- 平台 Hero 区
- 端点运行状态徽章
- 统计卡片
- 核心能力入口卡片
- 推荐工作流
- 架构概览

#### 主要交互

- 点击入口卡片进入对应模块
- 根据当前状态查看数据源数量、映射数量、本体类数量、查询历史数量

#### 依赖接口

- `GET /api/v1/datasources`
- `GET /api/v1/mappings`
- `GET /api/v1/sparql/history`
- `GET /api/v1/sparql/endpoint-status`
- `GET /api/v1/ontology`
- `GET /api/v1/ai/config`
- `GET /api/v1/ai/quick-questions`

### 7.2 登录页 `/login`

#### 目标

提供演示环境下的入口页与品牌展示。

#### 主要内容

- 平台介绍
- 登录表单
- 演示账号提示

#### 主要交互

- 输入任意非空密码即可进入首页
- 支持“跳过登录查看工作台”

#### 设计说明

- 当前页面不接真实认证
- 更偏演示和引导，而非安全入口

### 7.3 数据源管理 `/datasource`

#### 目标

管理数据库连接，并完成初步接入准备。

#### 功能点

- 查看数据源列表
- 新增数据源
- 删除数据源
- 测试连接
- 探测结构
- 全量 Bootstrap
- 查看生成文件路径

#### 输入项

- 数据源名称
- JDBC URL
- 用户名
- 密码
- 驱动类型

#### 输出项

- 连接状态
- 结构已探测状态
- Bootstrap 完成状态
- 本体、映射、属性文件路径

#### 成功标准

- 测试连接返回成功
- 可读取 schema
- 成功生成 `.ttl`、`.obda`、`.properties`

#### 依赖接口

- `GET /api/v1/datasources`
- `POST /api/v1/datasources`
- `DELETE /api/v1/datasources/{id}`
- `POST /api/v1/datasources/{id}/test`
- `GET /api/v1/datasources/{id}/schema`
- `POST /api/v1/datasources/{id}/bootstrap`

### 7.4 数据库概览 `/db-schema`

#### 目标

让用户理解数据库结构，并支持局部 Bootstrap。

#### 功能点

- 选择数据源
- 查看表数量、列数量、外键数量、主键数量
- 搜索表
- 查看表详情
- 查看主键、字段、外键、反向引用
- 选择若干表
- 选择是否包含依赖表
- 预览局部 Bootstrap 结果
- 生成局部 Bootstrap

#### 预览输出

- 请求表列表
- 实际纳入表列表
- 自动补入的依赖表
- 预估类
- 预估对象属性
- warning 信息

#### 生成输出

- 版本号
- 生成模式
- 本体文件路径
- 映射文件路径
- manifest 路径

#### 依赖接口

- `GET /api/v1/datasources`
- `GET /api/v1/datasources/{id}/schema`
- `POST /api/v1/datasources/{id}/bootstrap-preview`
- `POST /api/v1/datasources/{id}/bootstrap`

### 7.5 映射编辑 `/mapping`

#### 目标

让用户在结构化界面中维护 Ontop 映射规则。

#### 功能点

- 列出 `.obda` 文件
- 加载指定映射文件
- 查看 Prefix 声明
- 查看映射规则列表
- 查看规则摘要
- 新增映射规则
- 编辑映射规则
- 删除映射规则
- 保存映射
- 验证映射
- 重启端点

#### 映射规则展示字段

- Mapping ID
- Target
- Source

#### 验证结果

- 有效 / 无效
- 错误列表

#### 依赖接口

- `GET /api/v1/mappings`
- `GET /api/v1/mappings/content?path=...`
- `PUT /api/v1/mappings/content?path=...`
- `POST /api/v1/mappings/validate?path=...`
- `POST /api/v1/mappings/restart-endpoint`

### 7.6 SPARQL 查询 `/sparql`

#### 目标

提供语义查询调试与验证界面。

#### 功能点

- 编辑 SPARQL
- 执行查询
- 表格显示结果
- 查看 Ontop 重写 SQL
- 复制查询语句
- 查看历史查询
- 从历史回填查询

#### 数据展示

- 查询变量列
- 查询结果行
- SQL 文本
- 历史查询时间和结果条数

#### 异常处理

- Endpoint 不可用时显示错误
- 查询执行失败时清空结果表格并显示错误消息

#### 依赖接口

- `POST /api/v1/sparql/query`
- `POST /api/v1/sparql/reformulate`
- `GET /api/v1/sparql/history`

### 7.7 AI 助手 `/ai-assistant`

#### 目标

降低 SPARQL 使用门槛，让用户通过自然语言访问语义层。

#### 功能点

- 输入自然语言问题
- 使用快捷问题发起提问
- 流式展示处理过程
- 展示生成的 SPARQL
- 展示 SQL 重写结果
- 展示原始查询结果
- 展示最终中文答案

#### 交互流程

1. 用户输入问题
2. 页面新增用户消息
3. 页面新增“处理中”助手消息
4. 通过 SSE 接收中间步骤
5. 最终展示回答

#### 步骤面板

- SPARQL 查询
- SQL
- Results

#### 当前实现说明

- 快捷问题为页面内固定数组
- 尚未与设置页的快捷问题配置打通

#### 依赖接口

- `GET /api/v1/ai/query?question=...`

### 7.8 本体可视化 `/ontology`

#### 目标

以图谱和定义两种视角帮助用户理解本体结构。

#### 功能点

- 选择 `.obda` 文件
- 选择 `.ttl` 文件
- 关系图谱模式
- 本体定义模式
- 节点选中
- 缩放、自适应

#### 图谱模式

根据 `.obda` 提取：

- 类节点
- 类之间的关系边
- 属性字段摘要

适合：

- 快速理解类与关系结构
- 演示业务域结构

#### 定义模式

根据 `.ttl` 展示：

- 本体元数据
- Classes
- Object Properties
- Data Properties
- SHACL Constraints

适合：

- 校验本体定义完整性
- 查看约束表达

#### 依赖接口

- `GET /api/v1/mappings`
- `GET /api/v1/mappings/content?path=...`
- `GET /api/v1/ontology`
- `GET /api/v1/ontology/content?path=...`

## 8. 功能流程设计

### 8.1 主流程一：从数据库到本体

1. 在数据源管理页创建数据源
2. 测试连接
3. 读取 schema
4. 在数据库概览页选择全量或局部范围
5. 生成本体和映射
6. 在映射编辑页修正规则

### 8.2 主流程二：从映射到查询

1. 在映射编辑页验证映射
2. 重启 Endpoint
3. 在 SPARQL 页执行查询
4. 查看 SQL 重写
5. 确认结果正确

### 8.3 主流程三：从查询到 AI 问答

1. 在 AI 设置页配置模型
2. 在 AI 助手页输入自然语言问题
3. 查看自动生成的 SPARQL
4. 查看 SQL 和结果
5. 获取最终回答

## 9. 功能与接口映射

| 功能 | 前端页面 | 后端接口 | 核心依赖 |
|---|---|---|---|
| 数据源管理 | `/datasource` | `/datasources*` | SQLite, Ontop CLI |
| Schema 浏览 | `/db-schema` | `/datasources/{id}/schema` | Ontop CLI |
| 局部 Bootstrap | `/db-schema` | `/bootstrap-preview`, `/bootstrap` | Ontop CLI, 文件系统 |
| 映射编辑 | `/mapping` | `/mappings*` | OBDA Parser |
| 映射验证 | `/mapping` | `/mappings/validate` | Ontop CLI |
| 端点重启 | `/mapping`, TopBar | `/mappings/restart-endpoint` | Ontop Endpoint |
| SPARQL 查询 | `/sparql` | `/sparql/query`, `/sparql/reformulate` | Ontop Endpoint |
| AI 问答 | `/ai-assistant` | `/ai/query` | LLM, Ontop Endpoint |
| 本体定义查看 | `/ontology` | `/ontology/content` | TTL Parser |
| AI 配置 | `/settings` | `/ai/config` 等 | SQLite, LLM |
| 系统状态 | `/system` | `/config`, `/health` | FastAPI, Ontop Endpoint |

## 10. 功能边界与约束

### 10.1 当前约束

- 仅支持单实例工作台模式
- 无真实用户体系
- 无并发任务队列
- 无文件版本回滚界面
- AI 助手依赖外部 LLM 服务可用
- SPARQL 与 AI 查询依赖 Endpoint 已启动

### 10.2 使用前提

- Ontop CLI 路径正确
- 输出目录可写
- 目标数据库可连接
- 对应 JDBC 驱动可用
- LLM 服务地址正确

## 11. 异常场景设计

### 11.1 数据源异常

场景：

- JDBC URL 错误
- 用户名密码错误
- 数据库不可达

处理：

- 页面 toast 报错
- 后端返回连接失败消息

### 11.2 Bootstrap 异常

场景：

- Ontop CLI 执行失败
- 输出目录权限不足
- 选定表不存在或有歧义

处理：

- 后端返回 400
- 页面显示失败信息

### 11.3 查询异常

场景：

- Endpoint 未启动
- SPARQL 语法错误
- AI 生成了无效 SPARQL

处理：

- SPARQL 页面显示错误文本
- AI 助手显示失败消息

### 11.4 AI 配置异常

场景：

- 模型发现失败
- API Key 错误
- Base URL 无法访问

处理：

- 设置页回退到预设模型或手工输入
- 保存不阻断页面继续编辑

## 12. 验收建议

### 12.1 核心验收项

- 能创建并保存数据源
- 能成功读取 schema
- 能生成 `.ttl`、`.obda`、`.properties`
- 能编辑并保存映射
- 能通过映射验证
- 能重启 Endpoint
- 能执行 SPARQL 并看到 SQL
- 能完成 AI 提问并返回答案
- 能显示本体图谱和 TTL 定义

### 12.2 页面验收项

- 页面加载正常
- 侧边栏高亮正确
- 顶栏端点状态可刷新
- 所有核心按钮能触发正确反馈
- 错误信息可见

- 数据接入
- 结构理解
- 映射生成
- 映射维护

- AI 交互
