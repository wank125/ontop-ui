# 本体可视化方案对比

## 当前方案：React Flow (@xyflow/react)

**采用理由：** 原生 React 集成、节点即 React 组件（可直接用 shadcn/ui）、内置 Controls/MiniMap/Background、dagre 层次布局、暗色主题无缝融合。

| 维度 | 评价 |
|------|------|
| React 集成 | ★★★★★ — 节点是 React 组件，状态管理自然 |
| 视觉效果 | ★★★★ — 默认美观，自定义节点支持渐变/阴影/动画 |
| 交互体验 | ★★★★★ — 拖拽/缩放/框选/minimap 开箱即用 |
| 布局算法 | ★★★ — 需外部 dagre/elk 辅助，非内置 |
| 图论分析 | ★★ — 不内置图算法，需自行实现 |
| 大规模数据 | ★★★ — 数百节点流畅，数千节点需优化 |
| 学习曲线 | ★★★★ — API 简洁，文档完善 |
| 维护状态 | ★★★★★ — xyflow 团队活跃维护，商业支持 |

## 备选方案一：Cytoscape.js

**定位：** 图论分析 + 可视化引擎，学术/生物信息领域广泛使用。

```bash
pnpm add cytoscape
```

**优势：**
- 内置 30+ 图论算法（最短路径、中心度、连通分量、聚类系数）
- 10+ 种布局算法（cose、circle、breadthfirst、dagre、euler、cola 等）
- 类 CSS 的样式系统，支持条件样式和状态转换动画
- 完善的事件系统（节点/边/画布的 tap、hover、drag 等事件）
- 丰富的插件生态（cytoscape-dagre、cytoscape-cola、cytoscape-cose-bloom）

**劣势：**
- 非 React 原生，需要用 useRef + useEffect 包装，React 状态同步有额外工作
- 视觉风格偏学术/工具化，需要较多 CSS 定制才能达到现代 UI 效果
- 节点渲染基于 Canvas/SVG，不能直接嵌入 React 组件
- 深色主题需手动配置所有样式属性

**适用场景：**
- 需要图论分析（路径查找、社区发现、中心度计算）
- 复杂的图布局需求（层次、力导向、同心圆等混合）
- 学术研究、生物信息、网络分析项目

**集成代码示例：**
```typescript
import cytoscape from 'cytoscape';
import dagre from 'cytoscape-dagre';

cytoscape.use(dagre);

const cy = cytoscape({
  container: document.getElementById('cy'),
  elements: [/* nodes and edges */],
  style: [
    {
      selector: 'node',
      style: {
        'background-color': '#6366f1',
        'label': 'data(label)',
        'color': '#f3f4f6',
        'text-valign': 'center',
        'text-halign': 'center',
        'border-width': 1,
        'border-color': '#8b5cf6',
      },
    },
    {
      selector: 'edge',
      style: {
        'width': 1.5,
        'line-color': '#6b7280',
        'target-arrow-color': '#6b7280',
        'target-arrow-shape': 'triangle',
        'curve-style': 'bezier',
      },
    },
  ],
  layout: { name: 'dagre', rankDir: 'TB' },
});
```

## 备选方案二：Vis.js (vis-network)

**定位：** 通用网络可视化库，物理模拟动画是其最大特色。

```bash
pnpm add vis-network vis-data
```

**优势：**
- 物理引擎模拟动画效果出色（弹簧-电荷模型），节点布局自然
- 大规模数据表现好（数千节点），支持聚类简化显示
- 节点支持图片/图标/HTML 自定义
- 内置交互功能丰富：拖拽、缩放、选中、右键菜单
- 开箱即用，配置简单

**劣势：**
- 非 React 原生，需要 useRef 包装，React 状态同步繁琐
- 不内置图论算法
- 样式控制粒度较粗，无法做精细的节点内部布局
- 库体积较大（vis-network ~200KB gzipped）
- 维护节奏不稳定，社区活跃度下降

**适用场景：**
- 需要物理模拟的动态图效果
- 大规模图数据的快速浏览（数千节点）
- 原型快速搭建，对视觉精细度要求不高

**集成代码示例：**
```typescript
import { Network } from 'vis-network';

const container = document.getElementById('network');
const data = {
  nodes: [
    { id: 1, label: 'dim_store', color: { background: '#6366f1', border: '#8b5cf6' } },
    { id: 2, label: 'dim_employee', color: { background: '#6366f1', border: '#8b5cf6' } },
  ],
  edges: [{ from: 2, to: 1, label: 'ref-store_id', arrows: 'to' }],
};
const options = {
  physics: { stabilization: false, barnesHut: { gravitationalConstant: -2000 } },
  nodes: { shape: 'box', font: { color: '#f3f4f6', size: 14 } },
};
new Network(container, data, options);
```

## 备选方案三：D3.js

**定位：** 底层数据可视化库，最大灵活性但开发成本最高。

```bash
pnpm add d3 @types/d3
```

**优势：**
- 最大灵活性，可以实现任何自定义可视化效果
- 强大的数据绑定和过渡动画系统
- 丰富的可视化类型（力导向图、树图、桑基图等）
- 社区庞大，示例丰富

**劣势：**
- 开发成本最高，代码量大（通常 3-5 倍于其他方案）
- 非 React 友好，D3 的命令式 DOM 操作与 React 声明式模型冲突
- 需要手动处理所有交互（拖拽、缩放、选中）
- 没有内置的 UI 控件（minimap、工具栏等需自行实现）
- 学习曲线陡峭

**适用场景：**
- 需要完全自定义的可视化效果
- 高度定制化的交互体验
- 数据可视化团队有 D3 经验

**集成代码示例：**
```typescript
import * as d3 from 'd3';

const simulation = d3.forceSimulation(nodes)
  .force('link', d3.forceLink(edges).id((d: any) => d.id))
  .force('charge', d3.forceManyBody().strength(-400))
  .force('center', d3.forceCenter(width / 2, height / 2));

const svg = d3.select('#graph');
const link = svg.append('g').selectAll('line').data(edges).join('line');
const node = svg.append('g').selectAll('circle').data(nodes).join('circle')
  .attr('r', 20).call(d3.drag()
    .on('start', dragstarted)
    .on('drag', dragged)
    .on('end', dragended));

simulation.on('tick', () => {
  link.attr('x1', (d: any) => d.source.x).attr('y1', (d: any) => d.source.y)
      .attr('x2', (d: any) => d.target.x).attr('y2', (d: any) => d.target.y);
  node.attr('cx', (d: any) => d.x).attr('cy', (d: any) => d.y);
});
```

## 选型建议

| 场景 | 推荐方案 |
|------|---------|
| React 项目 + ER 图/本体展示 | **React Flow** |
| 需要图论算法分析 | Cytoscape.js |
| 大规模图浏览 + 物理动画 | Vis.js |
| 完全自定义可视化 | D3.js |
